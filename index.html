구글 메일 자동 발송을 위한 구글 클라우드 콘솔 설정 안내

자동 첨부 메일 전송 기능을 작동시키기 위해 최초 1회 아래의 구글 API 클라이언트 키 발급 설정이 꼭 필요합니다.

1단계: Google Cloud Console 접속 및 프로젝트 생성
  1. Google Cloud Console에 구글 계정으로 로그인합니다.
  2. 화면 좌측 상단의 프로젝트 선택 목록을 누르고 **'새 프로젝트'**를 생성합니다. (프로젝트 이름 예: `Import-BL-Converter`)
2단계: Gmail API 활성화
  1. 프로젝트 대시보드 좌측 상단 메뉴에서 **'API 및 서비스' -> '라이브러리'**로 이동합니다.
  2. 검색창에 **`Gmail API`**를 검색하여 클릭한 후, **'사용'** 버튼을 눌러 활성화합니다.
3단계: OAuth 동의 화면 설정
  1. 좌측 메뉴에서 **'OAuth 동의 화면'** 탭으로 이동합니다.
  2. User Type을 **'외부 (External)'**로 선택하고 생성 버튼을 누릅니다.
  3. 앱 이름(예: `필증송부 변환기`)과 사용자 지원 이메일, 개발자 연락처 이메일을 적고 저장합니다.
  4. **'범위 (Scopes)'** 단계에서 **'범위 추가 또는 삭제'**를 클릭한 후, **`.../auth/gmail.send`** (메일 발송 권한) 범위를 검색해 체크하고 추가합니다.
  5. **'테스트 사용자'** 단계에서 메일을 발송할 본인의 Gmail 주소를 **'테스트 사용자 추가'**를 통해 등록합니다. (반드시 등록해야 로그인 가능)
4단계: 사용자 인증 정보 발급 (클라이언트 ID 및 API 키)
  1. 좌측 메뉴에서 **'사용자 인증 정보'** 탭으로 이동합니다.
  2. 상단의 **`+ 사용자 인증 정보 만들기`**를 눌러 **'API 키'**를 생성합니다. 생성된 API 키를 복사하여 프로그램 설정의 `구글 API Key`에 입력합니다.
  3. 다시 **`+ 사용자 인증 정보 만들기`**를 눌러 **'OAuth 클라이언트 ID'**를 생성합니다.
    • 애플리케이션 유형: **웹 애플리케이션**
    • 이름: 자유롭게 입력
    • **승인된 JavaScript 원본**: 로컬 테스트용 주소(`http://localhost`) 및 깃허브 웹 주소(`https://a90894048-creator.github.io`)를 추가합니다.
  4. 생성 후 화면에 나타나는 **OAuth 클라이언트 ID** 문자열을 복사하여 프로그램 설정의 `구글 Client ID`에 입력합니다.
5단계: 로그인 연동 및 발송 시작
  1. 키를 입력하고 설정이 자동 저장되면 설정란의 **`구글 계정 인증하기`** 버튼이 활성화됩니다.
  2. 인증 버튼을 누르고 구글 로그인을 진행하여 앱 권한 승인 창에서 `계속` 및 `허용`을 선택합니다. (보안 경고 팝업이 뜨면 '고급' -> '이동'을 선택해 넘어갑니다.)
  3. 인증 상태가 **'인증 완료'** 배지로 바뀌면 메일 전송 및 일괄 발송 기능이 완전 자동으로 작동합니다.
